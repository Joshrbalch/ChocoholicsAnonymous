Glossary

1. Member: Person who pays monthly fees for consultations and treatments with health care professionals, namely, dietitians, internists, and exercise experts.
2. Provider: Health care professional that provides services to members.
3. Member card: Plastic card embossed with the member’s name, nine-digit member number, and magnetic strip on which information is encoded.
4. ChocAn Data Center: Computer/Group of Computers used to verify members, accounting procedures, and operations within ChocAn.
5. Provider Directory: Alphabetically ordered list of service names and corresponding service codes and fees
6. Manager: Given Summary reports and can request reports each week
7. Report:
a. Weekly email sent out to members that details the list of services provided to that member, sorted in order of service date.
b. Weekly email sent out to providers containing the list of services he or she provided to ChocAn members.
8. Summary Report: List that shows every provider to be paid that week, the number of consultations each had, his or her total fee for that week, the total number of providers who provided services, the total number of consultations, and the overall fee total are printed
9. Interactive Mode: Mode in which operators use ChocAn Data Center to add, delete, and update members and providers
10. Terminal: Interface for providers and operators to perform actions
11. Record: An account of all things that happen either for a member or provider.
12. Acme Account Services: Third-party service that is responsible for financial procedures. They also update the ChocAn data center with membership records every evening
13. Operator: Person who can add members and providers, can also delete members and provides, and can update details for both
14. Service code: Designated 6 digit code that is used to identify different services/sessions within the ChocAn system. Eg: 598470 is the service code for “session with dietician”.
15. Electronics Funds Transfer: Process of moving money from one bank account to another using computer-based technology.

Use Cases

Member Validation:

Use Case: Member Validation
Context: Checks membership through inputing nine-digit membership number into system, then dialing ChocAn Data Center, validating membership
Actors: Provider
Main Success Scenario:
1. Provider swipes member's card
1.1. System reads nine-digit member number into a variable
2. System dials ChocAn Data Center
3. System displays the word "Validated"
Extensions:
3a. If member is not validated, the system displays reason for invalidity
3b. If a member has not paid membership fees for at least a month, there member status will be set to suspended

Billing Procedure:

Use Case: Billing Procedure
Context: A Provider uses this procedure to bill ChocAn for a service provided to a Member.
Actors: Provider
Main Success Scenario:
1. Provider performs the Member Validation Use Case
2. Provider enters the date of service in format (MM-DD-YYYY)
3. Provider uses Provider Directory to look up the appropriate service code.
4. Provider enters in service code
5. ChocAn Software writes a record to disk, including the following:
Current date and time (MM-DD-YYYY HH:MM:SS),
Date service was provided (MM-DD-YYYY),
Provider number (9 digits)
Member number (9 digits)
Service code (6 digits)
Comments (100 characters) (optional)
6. Software looks up and displays the fee to be paid for the service and displays it on the Provider's terminal
7. Provider totals amount to be paid at end of week to verify ChocAn's total
Extensions:
4.a If Provider has entered a nonexistent code, an error message is displayed on their terminal
4.b If code is accepted, Provider may also enter comments about the service provided

Manager Request Accounting Procedure:

Context: Clock automatically creates a report outlining what payments must be made weekly every Friday at midnight
